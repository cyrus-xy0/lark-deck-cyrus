# Documentation

The runtime source stays at the repository root in `skills/`, `server/`,
`scripts/`, `config/`, `knowledge/`, and `library/`. Supporting documents live
here:

- `ops/` - install, packaging, cloud-agent deployment, and operations guides.
- `product/` - product direction, business rules, design system, and library maintenance.
- `development/` - contributor workflow and development notes.

Root-level `INSTALL.md`, `CLOUD_AGENT.md`, and `CONTRIBUTING.md` are compatibility
pointers for external links and common GitHub conventions.
